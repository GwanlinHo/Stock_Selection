#  選股系統開發紀錄 (Development Log)

## [2026-04-20] 系統效能強化與格式重構 (重大更新)

### 1. 篩選門檻收緊 (Filter Tightening)
*   **優化目標**: 提升選股含金量，並大幅減少 API 配額消耗。
*   **變更內容**:
    *   **L2 成交量門檻**: 由原本的 1 張 (1,000 股) 提高至 **1,000 張 (1,000,000 股)**，有效排除低流動性殭屍股。
    *   **L1 月線斜率門檻**: 將 MA20 斜率要求由 0.1% 提高至 **0.5%**，確保標的具備明確上升趨勢。
    *   **結果**: 海選入圍數量由 600+ 檔大幅下降至 ~250 檔，最終精選池維持在 **50 檔以內**。

### 2. 斷點續傳機制 (Resumable Execution)
*   **實作內容**: 
    *   精煉階段（L3/L4）啟動前會主動偵測 `candidates.json` 進度。
    *   自動跳過今日已成功抓取數據的標的，節省 FinMind API 配額。
    *   增加「每處理 10 檔自動存檔」機制，確保程序中斷時資料不流失。

### 3. 數據品質與邏輯修正
*   **營收數據 (YoY)**: 將營收抓取天數由 120 天擴大至 **450 天**，確保滿足 L4 計算 YoY（年增率）所需的 13 個月對照期，解決了營收數據顯示為 0 的問題。
*   **清理機制優化**: 
    *   快取清理改用檔案修改時間 (`mtime`) 判斷。
    *   新增 **週報自動清理**: 自動移除超過 30 天的舊 Markdown 報告，保持資料夾整潔。

### 4. 報告格式統一 (Standardized Format)
*   **佈局調整**:
    1.  **AI 深度分析** 直接置頂，提供即時市場洞察。
    2.  新增 **篩選漏斗統計**: 透明化呈現各階段（價量 -> 籌碼 -> 營收）過濾出的標的數量。
    3.  **精選池表格收緊**: 僅列出「全關卡通過 (L4)」的標的，不再顯示未達標之候選股。
